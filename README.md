##  시작하기

### 서버 연결 및 모델 설정

1. **추론 서버 선택**: 드롭다운에서 서버 선택 (inference_server, remote_server, local, 직접 입력)
2. **프로토콜 선택**: gRPC 또는 HTTP 선택
3. **서버 연결 확인**: "서버 연결 확인" 버튼 클릭
4. **모델 선택**: 연결 후 사용할 모델과 버전 선택
5. **농장코드 입력**: 저장 폴더명 입력 (예: FARM001)

### 영상 선택 및 시작

1. **저장 경로 선택**: YOLO Bbox Label 및 Frame 이미지를 저장할 출력 폴더 선택
2. **영상 모드 선택**: 라디오 버튼을 통해 모드 선택 후 진행 가능
   - **단일 영상**: 선택한 하나의 영상에 대해서만 진행
   - **폴더 모드**: 폴더 내 모든 영상을 자동으로 순회 처리
3. **영상 선택**: 단일 영상 모드에서는 파일 선택, 폴더 모드에서는 폴더 선택
4. **시작**: "▶ 시작" 버튼 클릭

##  어노테이션 시스템

### 메모리 기반 일괄 저장

어노테이션 입력 시 즉시 파일에 저장하지 않고, 메모리 스택에 Frame 이미지 및 Label을 저장하고 종료시 일괄 출력 전환합니다.

**작동 방식:**
1. 어노테이션 키 입력 시 프레임 이미지와 추론 결과를 메모리에 저장
2. 영상 종료(`q` 키) 또는 다음 영상(`0` 키) 이동 시 일괄 저장


### 라벨 클래스

| 키 | 클래스 | 저장 폴더 |
|----|--------|-----------|
| `l` | Lying (기립/누워있음) | `stand_lying/` |
| `s` | Farrow_Start (분만 시작) | `farrow_start/` |
| `f` | Farrowing (분만 중) | `farrowing/` |
| `e` | Farrow_End (분만 종료) | `farrow_end/` |
| `w` | Womb (자궁) | `womb/` |
| `n` | Nothing (없음) | `nothing/` |
| `h` | Hand (손) | `hand/` |

### 롤백(Undo) 기능

**키**: `Backspace` (백스페이스)

마지막에 입력한 어노테이션을 취소할 수 있습니다. 메모리에서 제거되며, 저장 카운트도 자동으로 감소합니다.
메모리 스택이 비워질 때까지 어노테이션을 삭제 할 수 있습니다.

**사용 예:**
- 실수로 잘못된 키를 입력해 클래스가 잘못 부여됐을 경우, 백스페이스를 통해 어노테이션 제거 후 다시 부여
- 여러 개의 어노테이션을 순차적으로 롤백 가능

##  영상 재생 제어

### 무한재생

영상이 마지막 프레임에 도달하면 자동으로 처음부터 반복 재생됩니다. 
사용자가 직접 종료(`q` 키)하거나 다음 영상으로 이동(`0` 키)할 때까지 계속 재생됩니다.
**수정 전 : 영상 마지막 프레임 도달시 completed 플래그를 반환하고 종료**
**수정 후 : Q 혹은 0번 숫자패드를 통해 종료 및 이동하는 경우에만 진행률 카운트 업데이트**

### 프레임 이동

영상을 프레임 단위로 정확하게 탐색할 수 있습니다.

| 키 | 동작 |
|----|------|
| `4` (←) | 1 프레임 전으로 이동 |
| `6` (→) | 1 프레임 후로 이동 |
| `7` (Home) | 60 프레임 전으로 이동 |
| `9` (Page Up) | 60 프레임 후로 이동 |
| `3` (Page Down) | 24 프레임 전으로 이동 |
| `1` (End) | 24 프레임 후로 이동 |

### 일시정지/재생

- **키**: `Space` (스페이스) 또는 `5` (숫자키패드)
- 일시정지 상태에서 프레임 이동 가능
- 프레임 이동 입력 감지시 영상 자동 정지 후 프레임 이동 구현 예정

### 화면 설정

- **전체화면**: `+` (플러스)
- **화면 크기 조정**: `-` (마이너스)
  - 1280x720 ↔ 640x480 전환
- **라벨 표시 토글**: `Enter` (엔터)
  - 추론 결과 바운딩 박스 표시/숨김

##  폴더 모드

폴더 모드는 여러 영상을 자동으로 순회 처리하는 기능입니다.

### 주요 기능

1. **자동 영상 탐색**: 폴더 내 모든 영상 파일(.mp4, .avi, .mov) 자동 탐색
2. **진행 상황 추적**: CSV 파일로 완료된 영상 기록
3. **미완료 영상 필터링**: 다음 실행 시 완료된 영상 자동 제외
4. **진행 상황 표시**: UI에서 전체/완료/남은 영상 수 표시

### 진행 상황 CSV

- **위치**: `{저장경로}/{농장코드}/progress_list.csv`
- **기능**: 완료된 영상명을 기록하여 재실행시 기록된 영상을 제외하고 진행
- **자동 기록**: 첫 어노테이션이 메모리에 저장될때 CSV에 영상명 기록

### 폴더 모드 사용법

1. 영상 모드에서 "폴더 모드" 선택
2. 영상이 포함된 폴더 선택
3. "▶ 시작" 클릭
4. 각 영상 처리 후:
   - `0` 키: 다음 영상으로 자동 이동
   - `q` 키: 전체 작업 종료

##  OpenCV 오버레이 

영상 재생 중 OpenCV 창에 다음 정보가 표시됩니다.

### 프레임 정보 (1행)
- **프레임 위치**: `{현재프레임}/{전체프레임}` 형식
- **평균 추론 시간**: 초당 처리 속도 표시

### 어노테이션 통계 (2행)
- **Check**: 처리된 프레임 수
- **S/F/E/N/W/h**: 각 클래스별 저장된 어노테이션 개수

### 상태 메시지 (3행)
- 현재 작업 상태를 텍스트로 표시 (예: "Start Labeling", "STOP", "Looping Video")

### 마지막 어노테이션 정보 (4행)
- **Latest**: 마지막에 기록한 어노테이션의 프레임 번호와 클래스를 표시, 어노테이션이 없으면 "Latest: None" 표시

### Pending 정보 (5행)
- **Pending**: 해당 영상내 기록된 메모리 어노테이션 카운트
- 영상 종료 또는 다음 영상 이동 시 일괄 저장됨

### 프레임 라벨 정보 (5행)
- **Mark**: 해당 영상내 현재 프레임에 기록된 클래스 정보
- 프레임 이동시에도 해당 프레임에 남아 표시되며, 확인을 용이하게 하기 위함

## 🛡️ 예외처리

### 예기치 못한 종료 대비

영상 파일 읽기 실패 등 예기치 못한 종료 상황에서도 메모리에 남아있는 어노테이션을 자동으로 저장합니다.

**동작:**
- 영상 읽기 실패 시 자동으로 `save_annotation_list()` 호출

**참고:**
- 강제종료(Ctrl+C, 창 닫기)는 Python 예외 처리로 이 코드까지 도달하지 못할 수 있습니다
- 하지만 영상 파일 읽기 실패 같은 예기치 못한 종료는 처리 가능합니다

## 📂 저장 구조

출력은 다음 구조로 저장됩니다:

```
{저장경로}/
└── {농장코드}/
    ├── stand_lying/      # l 키
    ├── farrow_start/     # s 키
    ├── farrowing/        # f 키
    ├── farrow_end/       # e 키
    ├── womb/            # w 키
    ├── nothing/         # n 키
    ├── hand/            # h 키
    └── progress_list.csv # 폴더 모드 진행 상황
```

각 폴더에는 다음 형식으로 저장됩니다:
- **이미지**: `{영상명}_{프레임번호}.jpg`
- **라벨**: `{영상명}_{프레임번호}.txt` (YOLO 형식)

## ⚠️ 주의사항

- 영상당 약 30개 어노테이션 기준 약 180MB의 RAM을 점유합니다.
- 영상당 pending 카운트를 40 이내로 잡으시는 것을 권장합니다.

| pending 수 | 메모리 사용 증가량 (MB) | 누적 증가량 (MB) |
| --------- | --------------- | ----------- |
| 0         | -               | -           |
| 10        | **12.3**        | **12.3**    |
| 30        | **25.5**        | **37.7**    |
| 70        | **29.2**        | **66.9**    |
| 100       | **110.8**       | **177.7**   |
| 130       | **41.0**        | **218.7**   |
| 170       | **28.4**        | **247.1**   |
| 200       | **28.4**        | **275.5**   |
| 250       | **48.7**        | **324.2**   |

메모리 점유율이 MB 단위라서,
일반적으로 100 ~ 400 프레임 길이의 영상을 매 프레임마다 모두 어노테이션 해도 문제가 없으나,  
다른 프로세스에 의해 메모리 대부분이 점유되어 가용 가능한 공간이 없다면,
pending 카운트를 높일 경우 프로그램이 Dump 될 수 있습니다.

총 증가량 (0 → 250 pending): ≈ 324 MB  
1장당 평균: 324 ÷ 250 = 약 1.30 MB  
10장당 평균: ≈ 13.0 MB  

**pending 70 → 100 구간에서 메모리 사용량이 급증한 이유**
메모리를 버퍼 단위로 제어하기 때문입니다.  
사용중인 장비에 따라 다르나, pending 70까지는 초기 할당한 버퍼 내에서 저장이 가능하지만,  
기존 버퍼 용량이 초과되면 버퍼를 해제 후 더 많은 용량으로 재할당하기 때문에 메모리 점프가 발생 할 수 있습니다.
